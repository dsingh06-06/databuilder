Core Features

Everything in
one file

No backend, no database server, no installation. Download once, run forever.

⊞
Visual Table Builder
Design tables with 22 field types including calculated, cross-table aggregate, and auto-timestamp. Set required, unique, indexed flags. Drag to reorder fields.
⊞∑
Cross-Table Aggregates
Link fields across tables. Automatically compute Sum, Count, Average, Min, Max from another table's records — filtered by a matching key. Powers live stock balance.
∑
Calculated Fields
Write any JavaScript expression as a formula. Total In - Total Out, price * qty, name + ' ' + surname. Space-in-name fields fully supported.
🕐✓
Auto Timestamp
Fields that auto-fill on Create, Update, or both. No manual date entry needed. Displayed as formatted local date-time in data view.
⚠
Alert Thresholds
Set a minimum value on any numeric field. Rows glow red when below threshold, amber when within 50% above it. Perfect for low-stock monitoring.
📊
Customisable Dashboard
Add any table's metric as a KPI widget card — choose aggregate, field, colour, and label. Reorder widgets. Persists across saves.
▾→⊞
Live Dropdown from Table
Link a dropdown field's choices to another table's column. New products added to Products immediately appear in Stock In / Stock Out dropdowns.
↺
Auto-Sync & Recalc
When a new record is saved, all linked tables update automatically. Use ↺ Recalc button to force-refresh all cross-table values, calculated fields and timestamps.

Quick Start Guide

Step-by-step
setup guides

📦 Setup Stock Inventory
1
Create table Products with fields: Product (Text, Unique), Category (Dropdown), Unit (Dropdown), Reorder Level (Number, Alert Threshold).
2
Create Stock In with fields: Product (Dropdown, From Table → Products.Product), Quantity (Number), Date, Supplier.
3
Create Stock Out with same structure as Stock In plus a Reason dropdown (Sale, Damaged, Expired).
4
Create Stock Balance. Add Total In — type Cross-Table Agg → Stock In → Quantity → Sum → Match: Product = Product.
5
Add Total Out same way pointing to Stock Out. Add Balance as Calculated field: Total In - Total Out.
6
Or skip all this — load sample_inventory.json via Import → JSON to get a fully wired workspace instantly.
⊞∑ Cross-Table Aggregate Setup
1
In the target table, add a new field. Select type Cross-Table Agg.
2
In Field Properties: choose Source Table (e.g. Stock In). The Source Field dropdown auto-populates.
3
Choose Aggregate: Sum / Count / Average / Min / Max.
4
Set Match Field (in source table) = This table's link field to filter by row. Leave blank to aggregate all rows.
5
The field will auto-compute on every save and on ↺ Recalc. Add it to a Dashboard widget for a live KPI card.
📊 Customise Dashboard
1
Click ⊞ Dashboard button in the header, or the Customize button on the dashboard view.
2
Select Table, Field, Aggregate (Sum/Count/Average/Min/Max), and a Label for the card.
3
Pick a colour (Cyan, Purple, Green, Amber, Red). Click + Add.
4
Use ↑↓ to reorder widgets or ✕ to remove them. Widgets persist across all save methods.
🚀 Export Data Entry App
1
Click 🚀 Export Data Entry App in the header. The app must be served via HTTP (VS Code Live Server, or any web server) — not opened directly by double-click.
2
The exported file contains all your tables, records, and dashboard widgets. Table Builder, Form Manager, and designer tools are stripped out.
3
Alternative: Since the code is section-tagged, manually delete §HTML-08, §HTML-09, §HTML-11, designer modals, and set IS_DATA_ENTRY_ONLY = true.
4
Share the single HTML file. Recipients open it in any browser — no installation, no server needed to use the runtime version.

FAQ

Common questions

Where is my data stored?
All data is stored locally in your browser's IndexedDB (with localStorage fallback). Nothing is uploaded to any server. If you use Ctrl+S with a pinned file, data is written directly to your disk. You can also export to JSON or Excel at any time for portable backups.
What happens if I clear my browser data?
IndexedDB and localStorage will be cleared. Always export a JSON or Excel backup before clearing browser data. The Ctrl+S pinned .xlsx file on your disk is unaffected.
Why does Export Data Entry App require a web server?
The export uses fetch(window.location.href) to read the original source file. Browsers block fetch() on file:// URLs for security reasons. Use VS Code's Live Server extension, Python's python -m http.server, or any static file server. The exported runtime app can be opened directly by double-click without a server.
Can I use this for stock inventory without setting it up manually?
Yes — load the included sample_inventory.json file via Import → JSON. It contains Products, Stock In, Stock Out, and Stock Balance tables with fully wired cross-table aggregates, 8 products, sample transactions, and 4 dashboard widgets pre-configured.
How do I change the currency symbol from $ to ₹?
In the JavaScript section §JS-23 (formatCellValue function), find '$' in the currency case and change it to '₹'. It's a single character change.
Can the Stock Balance update automatically when I add a new entry?
Yes. When you save a new record in Stock In or Stock Out, the app automatically recalculates all cross-table aggregate fields in linked tables (like Stock Balance) and refreshes the view. If values look stale, use the ↺ Recalc button in the data toolbar to force a full recompute.
Can I handle 1 million records?
Not comfortably. This app is browser-based and stores everything in memory. It performs smoothly up to ~5,000 records per table, usably to ~20,000, and will lag beyond that. For millions of records you'd need a backend database (PostgreSQL, MySQL) with this app as a frontend — a fundamentally different architecture.
Can I edit the HTML code directly?
Yes — the entire app is one HTML file with section markers throughout (§CSS-01 through §JS-43, §HTML-01 through §MODAL-13). Open in any text editor, find the section, edit, save. Each section is clearly labelled with what it controls.
